# 对话分析提示词模板

## 两阶段调用架构

本系统采用两次LLM调用的架构:
- 第一次调用(分析阶段): 解析对话,生成结构化分析结果,触发RAG检索
- 第二次调用(生成阶段): 结合分析结果+检索到的知识,生成最终回复建议

---

## 回复质量自检规则 (附加在生成提示词末尾)

```
生成回复前,用以下checklist自检每个回复选项:

[ ] 像真人微信聊天吗？去掉任何书面语和套话
[ ] 长度合适吗？超过40字就太长了,砍掉
[ ] 有没有跪舔嫌疑？检查是否过度讨好、无条件顺从
[ ] 有没有油腻感？检查是否像情感博主的万能模板
[ ] 对话能继续吗？这条回复是否给对方接话的空间
[ ] 符合当前温度吗？冷淡期不要突然过热,升温期不要浇冷水
[ ] 节奏对吗？如果她回复很慢,建议不要秒回

如果任何一项不通过,重新生成该选项。
```
