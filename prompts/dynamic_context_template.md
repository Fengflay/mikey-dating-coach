# 动态上下文注入模板

以下模板在运行时注入到系统提示词之后,根据用户的具体场景动态填充。

---

## 模板: 用户画像上下文

```
[用户画像]
- 用户ID: {{user_id}}
- 使用次数: {{session_count}}
- 常见问题模式: {{recurring_issues}}  // e.g. "需求感过强", "不会制造张力"
- 当前追求对象数: {{target_count}}
- 上次会话摘要: {{last_session_summary}}
```

## 模板: 对话历史上下文

```
[对话背景]
- 对象昵称: {{target_nickname}}
- 认识渠道: {{how_met}}  // e.g. "探探", "朋友介绍", "公司同事"
- 认识时长: {{days_known}}
- 是否见过面: {{met_offline}}
- 上次聊天时间: {{last_chat_time}}
- 历史分析记录:
{{#each previous_analyses}}
  - {{date}}: 阶段={{stage}}, 温度={{temperature}}, 关键发现={{finding}}
{{/each}}
```

## 模板: 知识库检索结果注入

```
[参考知识]
以下是与当前场景相关的专业指导,仅作为分析参考,不要直接引用或复述:

{{#each retrieved_knowledge}}
---
来源: {{source}}
场景: {{scenario}}
要点: {{key_points}}
---
{{/each}}
```

## 模板: 场景特殊指令

根据检测到的特殊场景,注入额外指令:

```
{{#if is_conflict}}
[特殊指令-冲突场景]
当前检测到冲突信号。优先安抚情绪,不要急于推进关系。分析冲突根源,判断是真生气还是撒娇式生气。如果是用户的错,直接告诉他应该道歉,但教他怎么道歉才不显得卑微。
{{/if}}

{{#if is_ghosting}}
[特殊指令-被冷落场景]
对方可能在逐渐失去兴趣或者在考验用户。重点分析是否应该继续投入。如果温度持续低于3,建议用户降低投入频率,转移注意力。不要给用户虚假希望。
{{/if}}

{{#if is_first_message}}
[特殊指令-破冰场景]
这是用户准备发送的第一条消息。开场白必须有记忆点,避免"你好"、"在吗"等无效开场。根据认识渠道定制开场策略。
{{/if}}

{{#if escalation_opportunity}}
[特殊指令-升级窗口]
检测到关系推进窗口。回复建议应包含至少一个选项引导线下见面或电话。不要让用户错过窗口。
{{/if}}
```
